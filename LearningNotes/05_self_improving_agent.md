# 🤖 第5课：让 AI 自己学会不犯错

> **难度**：⭐⭐⭐ 高级 | **阅读时间**：6分钟
> **一句话总结**：犯错 → 自动记日记 → 下次自动翻日记 → 不再犯同样的错。

---

## 先问你一个问题

你的 AI 助手今天犯了一个错误——比如用错了命令，搞坏了一个文件。

你帮它纠正了。

**一周后，同样的场景，它又犯了同样的错。**

为什么？因为它没有"记住"上次的教训。每次对话开始，它的脑子都是空白的。

---

## 人类是怎么学习的？

```
小明第一次骑自行车：
  摔了一跤 🤕
    ↓
  记住了："起步时要用力蹬，不要犹豫"
    ↓
  第二次骑：不犹豫了，稳了 ✅
    ↓
  再也不会在起步时犹豫了
```

**犯错 → 记住教训 → 下次不犯 = 学习**

---

## 教 AI 也这样学习

OpenClaw 支持两个特殊的"插件"，让 AI 拥有和人类一样的学习能力：

### 🔌 插件1：自动错误监控 (self-improving-agent)

**它干什么**：偷偷在后台看着 AI 的每一次操作。一旦发现错误，自动记下来。

```
AI 执行了一条命令
     │
     ▼
  成功了？→ 什么都不做，继续
     │
   失败了！
     │
     ▼
  自动记录到 ERRORS.md：
  ┌────────────────────────────────┐
  │ 📅 2026-03-24 01:30            │
  │ 🤖 Agent: David (程序员)       │
  │ ❌ 错误: npm install 失败       │
  │ 📝 原因: Node 版本不对         │
  │ ✅ 修复: 先执行 nvm use 22     │
  └────────────────────────────────┘
```

### 🔌 插件2：主动知识注入 (proactive-learning)

**它干什么**：每次 AI 开始工作前，自动翻"日记本"，把过去犯的错和学到的经验注入到 AI 的脑子里。

```
AI 收到一个新任务
     │
     ▼
  [自动注入] "提醒：上次 npm install 失败是因为 Node 版本不对。
              先执行 nvm use 22 再安装。"
     │
     ▼
  AI：好的！先切换 Node 版本 ✅
```

### 两个插件配合起来：

```
         犯错
          │
          ▼
   ┌──────────────┐
   │ self-improving │  ← 自动记录错误
   │   agent        │     写进 ERRORS.md
   └───────┬──────┘
           │
           ▼
      📓 ERRORS.md
      (错误日记本)
           │
           ▼
   ┌──────────────┐
   │  proactive    │  ← 自动读取日记
   │   learning    │     下次干活前注入
   └───────┬──────┘
           │
           ▼
     AI 不再犯同样的错 ✅
```

## 更高级：分阶段注入

AI 干不同阶段的活，需要的"经验"不一样：

| 阶段 | 注入什么 |
|------|---------|
| 📝 规划阶段 | "上次架构选型的经验" |
| 💻 编码阶段 | "上次代码里的Bug记录" |
| 🧪 测试阶段 | "上次容易漏测的场景" |

**不是把所有经验一股脑塞进去**（那样会太多太杂），而是**只给当前阶段相关的经验**。

就像你考数学的时候，只需要数学笔记，不需要语文笔记。

## Heartbeat：定时"复习"

除了犯错时自动记录，还有一个定时机制叫 **Heartbeat**（心跳）。

```
每隔 30 分钟：
  ┌────────────────────────────────┐
  │ 🫀 Heartbeat 触发              │
  │                                │
  │ AI 暂停手头的活，想一想：        │
  │                                │
  │ 1. 最近有没有犯过什么错？        │
  │ 2. 有没有学到什么新东西？        │
  │ 3. 有没有待办事项要做？          │
  │                                │
  │ → 把想到的写进日记              │
  └────────────────────────────────┘
```

就像老师每节课下课前说："同学们回顾一下这节课学了什么。"

## 共享学习：团队也能互相学

AI 团队里的不同成员，可以共享「日记本」：

```
💻 程序员 AI 犯了一个错：
  "部署时忘记设环境变量"
     │
     ▼ 写入共享日记本
     │
🌌 产品经理 AI 在写需求时自动看到：
  "提醒：部署时需要设置环境变量，
   上次漏了导致线上报错。
   请在需求文档里写明这一步。"
```

在 OpenClaw 里，这通过 `memorySearch.extraPaths` 配置实现——让不同 AI 可以读取同一个"经验库"。

```json
{
  "memorySearch": {
    "extraPaths": [
      "~/.openclaw/workspace-shared/.learnings/"
    ]
  }
}
```

## 📝 简单总结

| 机制 | 做什么 | 比喻 |
|------|--------|------|
| self-improving-agent | 犯错时自动记录 | 错题本 |
| proactive-learning | 干活前自动注入经验 | 考前翻笔记 |
| Heartbeat | 定时反思和复习 | 课后总结 |
| 共享学习 | 团队共享错题本 | 班级共享笔记 |

**四套机制组合起来 = AI 团队越用越聪明。**
